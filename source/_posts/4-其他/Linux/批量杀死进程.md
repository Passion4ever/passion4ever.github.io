{%note info%}
Linux 环境下批量杀死进程的一个 shell 脚本
{%endnote%}

<!-- more -->

脚本内容如下: 
```bash
#!/bin/bash

# 检查命令参数
if [ $# -lt 1 ]; then
  echo "必须提供包含关键字的进程名称作为参数"
  exit 1
fi

# 提取命令参数
keyword=$1
kill_processes=false
# 第二个参数为 1 时杀掉进程, 否则不杀
if [ "$2" = 1 ]; then
  kill_processes=true
fi

# 查找并处理进程
# `grep -v grep` 是过滤掉grep命令本身的进程
# 添加了 `grep -v $$` 来过滤掉脚本搜索关键字时的进程
processes=$(ps -ef | grep $keyword | grep -v grep | grep -v $$)   
if [ -z "$processes" ]; then
  echo "没有找到匹配的进程"
else
  echo "找到的进程："
  echo "$processes"

  if [ "$kill_processes" = true ]; then
    processes_to_kill=$(echo "$processes" | awk '{print "kill -9 "$2}')
    echo "杀死进程："
    echo "$processes_to_kill"
    echo "$processes_to_kill" | sh
  fi
fi
```
如何运行: `sh xxx.sh [你要搜索或杀死的进程关键词] [1 or 其他]`
第一个参数是必写参数, 第二个参数是可选的, 不写或者是非 1 表示不杀死进程只罗列出来.
